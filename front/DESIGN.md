# Groove Work — концепция UI «Expressive Glass»

Развитие Material 3 Expressive: к тональным палитрам OKLCH добавляются два
выразительных приёма — **фирменные градиенты** и **акрил** (полупрозрачность +
blur). Всё построено на токенах темы (`assets/tokens.css`) — концепция целиком
динамическая: меняется цвет темы в конструкторе → меняются градиенты и стекло.
Эталон реализации — сайдбар (`components/layout/AppSidebar.vue`).

## Три кита

1. **Градиент — знак «здесь и сейчас».** Горизонтальный градиент primary →
   светлый тон отмечает ЕДИНСТВЕННОЕ активное/главное на экране: активный пункт
   навигации, активную вкладку, primary-кнопку, FAB. Не фон, не декорация —
   максимум 1–2 градиентных акцента в поле зрения.
2. **Акрил — слой над контентом.** Всё, что «плавает» над страницей
   (сайдбар, нижняя навигация, поповеры, шторки, диалоги), — полупрозрачное
   стекло с blur: контент мягко просвечивает, иерархия слоёв читается без
   жёстких границ.
3. **Смысловая группировка.** Навигация и длинные списки делятся на секции с
   тонкими full-bleed разделителями и маленькими серыми подписями
   («Рабочие процессы», «Коммуникация с командой», «Управление и анализ»).

## Токены (Layer 4, tokens.css)

| Токен | Назначение |
|---|---|
| `--grad-primary` | активный пункт нав., активная вкладка, primary-кнопки, FAB |
| `--grad-primary-soft` | тинтованные плашки: карточка компании, карточка профиля, выбранные строки |
| `--grad-hero` | крупные hero-шапки разделов (диагональный, primary → tertiary) |
| `--acrylic-bg` | фон стеклянных поверхностей (пара к `--acrylic-blur`) |
| `--acrylic-bg-strong` | плотное стекло: sticky-шапки, бейджи на градиенте |
| `--acrylic-blur` | `backdrop-filter` стекла (не забывать `-webkit-`-префикс) |
| `--acrylic-border` | полупрозрачная обводка стеклянных поверхностей |
| `--acrylic-card-bg` | подложка карточек В ПОТОКЕ страницы — та же матовая плотность, что у `--acrylic-bg`, но БЕЗ blur |
| `--bgg-1..4` | слои пользовательского градиента фона (пишет theme.js из рецепта) |
| `--bg-grad-dim` | гаситель долей градиента фона в тёмной теме |

Тёмная тема и hex-фолбэк для старых Safari определены там же — компонентам
ничего проверять не нужно. Правило прежнее: никаких `#hex`/`rgba()` в
компонентах, только токены.

## Рецепты

**Градиент фона приложения.** Фон `.app-layout` — четыре радиальных «пятна» из
цветов темы поверх `--color-bg`. Включён по умолчанию, управляется в настройках
внешнего вида (тумблер + «Сгенерировать вариант»): theme.js хранит РЕЦЕПТ
(роль токена, позиция, размер, доля — localStorage `gw_bg_gradient`) и пишет
готовые слои в `--bgg-1..4`; цвета внутри — `color-mix` от `var(--color-*)`,
поэтому градиент сам следует за темой и тёмным режимом (`--bg-grad-dim`).
Именно этот градиент просвечивает сквозь всё стекло ниже — на нём и держится
акриловая концепция.

**Три сорта стекла — выбирать по тому, что окажется ЗА поверхностью:**
1. *Карточка в потоке* (за ней только фон страницы): `background: var(--acrylic-card-bg)` — без backdrop-filter, дёшево, можно десятками. За карточкой только гладкий градиент фона — blur его не изменил бы, поэтому тинта достаточно; исключение — крупные одиночные панели-каркасы (`.rail`, `.split-side`, `.split-main`, колонки настроек), им blur добавлен для честной матовости.
2. *Плавающая поверхность* (поповер, шторка, модалка, панель — под ней контент):
```css
background: var(--acrylic-bg);
backdrop-filter: var(--acrylic-blur);
-webkit-backdrop-filter: var(--acrylic-blur);
border: 1px solid var(--acrylic-border);
```
3. *Sticky-шапки* (контент прокручивается под ними): как п.2, но `--acrylic-bg-strong` — плотнее ради читаемости текста.

Модалки: базовый `AppDialog` и `.p-dialog` уже акриловые — внутренним секциям модалок фон не задавать (или прозрачный/полупрозрачный `color-mix`), чтобы не глушить стекло контейнера.

**Активный элемент навигации/вкладок:** фон `var(--grad-primary)`, текст
`var(--color-on-primary)`, иконка Material Symbols с `'FILL' 1`.
У hover-состояний градиента нет — только тинт
`color-mix(in oklch, var(--color-primary) 10%, transparent)`.

**Бейджи-счётчики** — тинтованные прямоугольные плашки (не «алярмные» красные
кружки): фон `color-mix(in oklch, var(--color-primary) 14%, var(--color-surface))`,
текст `var(--color-primary)`, тонкая primary-обводка, радиус `--radius-sm`.
Красный/error — только для настоящих тревог (питомец заболел → бейдж `!` на
`--color-error-container`).

**Тинтованные карточки-плашки** (компания, профиль): фон
`var(--grad-primary-soft)`, обводка
`color-mix(in oklch, var(--color-primary) 16%, transparent)`, радиус `--radius-lg`.

**Двухтоновый wordmark:** «Groove» — `var(--color-primary)`, «Work» —
`color-mix(in oklch, var(--color-primary) 40%, var(--color-primary-container))`.

## Паттерны экрана-списка

Единый язык разделов-списков (эталон — «Задачи»; так же собираются Реестры,
Календари, Пользователи, Компании). Всё глобальное живёт в `assets/main.css`
(блок «Паттерны экрана-списка») и `components/common/` — в компонентах раздела
остаются только его мелочи.

**Каркас (десктоп):** прозрачный тулбар сверху (страница лежит на «сиянии»
`.app-layout` — фон не дублировать) → под ним слева стеклянная рейка фильтров,
справа сетка/список стеклянных карточек. Мобильный (≤768px) — свой рисунок:
плотная шапка, вкладки `SegmentedTabs`, фильтры/сортировка — шторками.

**Тулбар:**
- Поиск — `components/common/SearchField.vue`: стеклянная пилюля, v-model,
  крестик очистки, Esc; проп `hotkey` регистрирует Cmd/Ctrl+K (фокус) и сам
  показывает kbd-хинт по платформе (`kbd`-проп/слот `hint` — переопределить).
- Переключатель вида — `SegmentedTabs` с `variant="glass"`: стеклянный
  контейнер, активная вкладка — тинтованная primary-пилюля (не градиент —
  это второстепенный контрол, градиент оставляем главной кнопке). Табы могут
  быть icon-only (`{ value, icon }`).
- Главное действие — `.btn-grad` (пилюля на `--grad-primary`), вторичное —
  `.btn-glass` (стеклянная пилюля).

**Рейка фильтров — глобальные классы:**
- `.rail` — стеклянная sticky-колонка 250px (`--acrylic-card-bg` +
  `--acrylic-border`, radius `--radius-xl`); отступы от краёв страницы задаёт
  раздел.
- `.rail-scroll` — прокручиваемая середина (низ с `.rail-reset` не уезжает).
- `.rail-section` + `.rail-label` — группа пунктов с мелкой dim-подписью
  (СОРТИРОВКА, УЧАСТИЕ, …).
- `.rail-item` — вертикальный пункт: высота 38px, radius `--radius-md`,
  иконка 20px + подпись 13.5px/600; hover — тинт primary 8%; `.active` —
  пилюля `color-mix(in oklch, var(--color-primary) 14%, var(--color-surface))`
  + текст/иконка primary (иконка заливается `FILL 1`).
- `.rail-badge` — счётчик у пункта (правый край); на активной пилюле
  автоматически инвертируется в primary/on-primary.
- `.rail-reset` — ghost-кнопка «Сбросить все» внизу (hover — тинт error).
- Selects/чипы внутри секций — обычные контролы раздела; в один пункт-стиль
  их не переодевать.

**Чипы статусов/меты:** `.chip-tint` (нейтральный: surface-high + text-dim)
и модификаторы `.chip-tint--primary|--error|--warning|--success` — фон
`*-container`, текст `on-*-container`, иконка 16px. Просрочено → `--error`,
«скоро срок» → `--warning`, отдел-заказчик → `--primary`.

**Кнопки:** `.btn-grad` — главная (градиент, hover `brightness(1.06)`);
`.btn-glass` — вторичная стеклянная; `.btn-soft-success` — мягкое
действие-запуск («В работу»: success-container + иконка play_arrow).

**Карточка списка:** стекло `--acrylic-card-bg` + `--acrylic-border`,
radius ~18px, ТЕНЕЙ ПОЧТИ НЕТ — только `--shadow-sm` + `translateY(-2px)`
на hover. Ряды: чип контекста + экшены → название (15px/700, clamp 2) →
мета-чипы → кнопка действия. Цветная карточка (личный тег) — тот же макет,
но фон `--tag-*-surface` вместо стекла. Канбан/список — те же стеклянные
поверхности, без отдельного языка.

### Мастер-детейл разделы

Разделы «список сущностей слева → содержимое выбранной справа» (Реестры,
Календари, Заметки/Ежедневник). Каркас — глобальные классы `.split-*` в
`assets/main.css` (конец блока «Паттерны экрана-списка»); в компонентах
разделов остаются только их внутренности.

- `.split-view` — корень раздела: ряд панелей во всю высоту, gap/padding 16px,
  без собственного скролла (скроллятся только внутренности панелей).
- `.split-side` — левая стеклянная колонка 300px (`--acrylic-card-bg` +
  `--acrylic-border`, radius `--radius-xl`), flex-колонка.
- `.split-side-head` — шапка: `.split-side-tile` (иконка-тайл 40px, скруглённый
  квадрат на `--color-primary-container`, иконка primary — table_view /
  calendar_month / note_stack) + `.split-side-title` (17px/750). У Заметок
  вместо тайла в шапке живёт переключатель хаба (`NotesHubTabs`).
- `.split-side-list` — прокручиваемый список; `.split-side-note` — dim-заметка
  («Загрузка…», «Реестры отсутствуют»).
- `.split-side-item` — пункт: `.split-item-tile` (мини-тайл 32px, тинт primary
  11%, иконка primary; на активном — `FILL 1`) + `.split-side-name`
  (14px/600, ellipsis). Активный — тинтованная пилюля
  `color-mix(primary 14%, surface)` + текст primary; hover — тинт 8%.
  Мета пункта (прогресс-бары ежедневников, владелец «Поделились») — свои
  элементы раздела внутри пункта.
- `.split-side-add` — ghost-пункт создания внизу списка (как `.rail-reset`,
  но primary: «Новый ежедневник»).
- `.split-main` — правая область: ОДНА большая стеклянная панель; внутренние
  тулбары/таблицы/сетки — на прозрачном/полупрозрачном фоне внутри неё
  (стекло на внутренних блоках не дублировать; sticky-шапки таблиц —
  `--acrylic-bg-strong` как обычно).
- Пустое состояние («ничего не выбрано») — `EmptyState` с `tone="soft"` и
  классом `.split-empty` (центр панели): иконка раздела в мягком круге 96px
  (`color-mix(surface 70%, transparent)` + `--shadow-sm`), заголовок 16px/700,
  dim-подпись.
- Мобайл (≤768px): `.split-side` скрыта (выбор — чипами/шторками раздела),
  `.split-main` — во весь экран без рамки; это тоже в глобальном медиа-блоке.

Паттерны раздела «Заметки» (переиспользовать при похожих задачах):
- *Плитка-стикер* (`NotesView`): карточка в потоке `--acrylic-card-bg` +
  `--acrylic-border`, radius ~18px, заголовок 15px/700 (clamp 2 строки),
  dim-превью текста (clamp 3), низ — дата с иконкой 15px.
- *Панель форматирования редактора* (`components/notes/NoteRichEditor.vue`):
  sticky-стекло `--acrylic-bg-strong` + blur (правило sticky-шапок), кнопки
  30px с тинт-hover и primary-active, группы через тонкие сепараторы; на
  мобиле — одна строка с горизонтальным скроллом. Палитра выделения текста —
  токены задач `--tag-*-surface` (никаких hex в документе — в TipTap-doc
  хранится `var(--tag-…)`).

## Осторожно

- Акрил требует непрозрачного контента ПОД текстом только через
  `--acrylic-bg-strong` — длинные тексты на слабом стекле не размещать.
- `backdrop-filter` дорог: не вешать на элементы, которых на экране десятки
  (строки списков, карточки сетки) — только на крупные одиночные слои.
- Градиент не для больших заливок фона и не для текста на нём мельче 12px.
